1️⃣ Cara Setting Port Aplikasi

Metode 1: Ubah di File app.py (Permanen)

Buka file app.py dan cari baris terakhir:

port = int(os.environ.get("PORT", 5000))

Ubah angka 5000 menjadi port yang diinginkan, misal 8080:

port = int(os.environ.get("PORT", 8080))

✅ Simpan file, restart aplikasi.

Metode 2: Via Environment Variable PORT

Jalankan dengan port dinamis tanpa mengubah kode:

# Windows CMD set PORT=8080 && python app.py # PowerShell $env:PORT=8080; python app.py # Linux/Mac Bash PORT=8080 python app.py

💡 Environment variable PORT akan override nilai default di app.py.

Metode 3: Via PM2 ecosystem.config.js

Atur port di file ecosystem.config.js:

env: { PORT: 8080 }

💡 Lihat panduan PM2 di bawah.

2️⃣ Cara Setup dan Jalankan Program

📋 Prasyarat

  • Python 3.8+ terinstal (python --version)
  • pip terinstal (pip --version)

⚙️ Langkah Setup

  1. Buka terminal/command prompt dan arahkan ke folder project:
    cd C:\Users\prime\Documents\Program_Test_Gudang
  2. Install dependencies (Flask + pytest):
    pip install -r requirements.txt
  3. Jalankan aplikasi:
    python app.py
  4. Buka browser dan akses:
    http://127.0.0.1:5000
  5. 💡 Tekan Ctrl+C untuk stop.

🧪 Menjalankan Test

python -m pytest tests/ -v

Menjalankan 50 unit test & integration test.

3️⃣ Cara Setup Virtual Environment (venv)

Virtual environment digunakan untuk mengisolasi dependensi Python proyek agar tidak tercampur dengan instalasi global.

🪟 Windows

# Buat virtual environment python -m venv venv # Aktifkan venv\Scripts\activate # Install dependencies di dalam venv pip install -r requirements.txt # Jalankan aplikasi python app.py # Nonaktifkan venv (setelah selesai) deactivate

🐧 Linux / Mac

# Buat virtual environment python3 -m venv venv # Aktifkan source venv/bin/activate # Install dependencies pip install -r requirements.txt # Jalankan python app.py # Nonaktifkan deactivate

🤖 Menggunakan venv dengan PM2

# Pastikan PM2 menggunakan interpreter dari dalam venv: pm2 start ecosystem.config.js \ --interpreter venv\Scripts\python # Windows pm2 start ecosystem.config.js \ --interpreter venv/bin/python # Linux/Mac

💡 Atau update interpreter di ecosystem.config.js.

4️⃣ Cara Setup dan Jalankan dengan PM2

🔧 Apa itu PM2?

PM2 (Process Manager 2) adalah process manager untuk Node.js, tetapi juga bisa menjalankan aplikasi Python. PM2 menjaga aplikasi tetap berjalan di background, melakukan auto-restart jika crash, dan menyediakan load balancing serta logging terpusat.

📥 Install PM2

PM2 membutuhkan Node.js. Install lewat npm:

npm install -g pm2

💡 Cek instalasi: pm2 --version

🚀 Menjalankan Aplikasi dengan PM2

Gunakan file ecosystem.config.js yang sudah disediakan:

cd C:\Users\prime\Documents\Program_Test_Gudang # Start aplikasi pm2 start ecosystem.config.js # Atau langsung tanpa file config pm2 start app.py --interpreter python --name stok-gudang

📋 Perintah Dasar PM2

pm2 status # Lihat status semua proses pm2 list # Sama dengan status pm2 logs stok-gudang # Lihat log aplikasi (realtime) pm2 logs stok-gudang --lines 100 # 100 baris terakhir pm2 stop stok-gudang # Stop aplikasi pm2 restart stok-gudang # Restart aplikasi pm2 delete stok-gudang # Hapus dari daftar PM2 pm2 monit # Monitor CPU/Memory (interaktif) pm2 show stok-gudang # Detail proses

🔄 Auto-start Saat Reboot (Production)

Agar aplikasi otomatis hidup saat komputer/server restart:

# Simpan daftar proses pm2 save # Generate startup script pm2 startup

Di Windows:

  • PM2 startup di Windows memerlukan pm2-installer atau service manager
  • Alternatif: buat Task Scheduler Windows untuk menjalankan pm2 start ecosystem.config.js
  • Atau gunakan nssm (Non-Sucking Service Manager) untuk daftarkan python app.py sebagai Windows Service
⚠️ Catatan: pm2 startup berfungsi penuh di Linux/Mac. Di Windows, gunakan alternatif di atas.

📝 Mengatur Port di ecosystem.config.js

module.exports = { apps: [{ name: "stok-gudang", script: "app.py", interpreter: "python", // Environment variable PORT akan override default di app.py env: { PORT: 5000 }, // Ganti port untuk mode production env_production: { PORT: 8080 }, watch: false, autorestart: true, max_memory_restart: "200M" }] }; # Jalankan dengan environment tertentu pm2 start ecosystem.config.js # pakai env pm2 start ecosystem.config.js --env production # pakai env_production

5️⃣ Menjalankan di Environment Terbatas

📡 5.1. Tanpa Koneksi Internet (Offline)

Jika server tidak memiliki akses internet, siapkan offline packages dari komputer yang terhubung internet:

Langkah 1: Download packages di komputer yang ada internet

# Buat folder untuk menyimpan packages mkdir offline_packages # Download semua dependencies beserta dependensinya pip download -r requirements.txt \ -d offline_packages

Langkah 2: Copy folder offline_packages ke server offline

Langkah 3: Install dari folder local di server offline

# Install tanpa internet pip install --no-index \ --find-links ./offline_packages \ -r requirements.txt # Atau install satu per satu pip install --no-index \ --find-links ./offline_packages \ flask pytest

🔒 5.2. Port Terbatas (Firewall)

Jika hanya port tertentu yang diizinkan oleh firewall/IT:

# Cari port yang tersedia (contoh: 8080, 3000, 8000) netstat -ano | findstr :8080 # Windows ss -tlnp | grep :8080 # Linux # Set port yang diizinkan set PORT=8080 && python app.py # Windows PORT=8080 python app.py # Linux # Atau via PM2 pm2 start ecosystem.config.js --env production
💡 Jika menggunakan reverse proxy (Nginx/Apache/IIS), cukup set port internal seperti 5000, lalu proxy dari port 80/443.

💾 5.3. Resource Terbatas (RAM/CPU Rendah)

Untuk VPS atau server dengan RAM minimal (512MB - 1GB):

  • Gunakan production WSGI server (lebih hemat dari Flask dev server):
# Install waitress (ringan, native Python, no C compiler needed) pip install waitress # Jalankan dengan waitress python -c " from waitress import serve from app import app import os port = int(os.environ.get('PORT', 5000)) serve(app, host='0.0.0.0', port=port) "
  • Matikan debug mode di app.py: app.run(debug=False, host="0.0.0.0", port=port)
  • Batasi memory PM2: max_memory_restart: "100M"
  • Gunakan SQLite (sudah default) — lebih ringan dari PostgreSQL/MySQL

🛡️ 5.4. Tanpa Hak Admin (No Admin/Sudo)

Jika tidak punya hak admin di server:

  • Python user install (--user flag):
# Install tanpa admin (hanya untuk user) pip install --user -r requirements.txt
  • PM2 lokal install (tanpa -g global):
# Install PM2 lokal di folder project npm install pm2 # Jalankan dari node_modules npx pm2 start ecosystem.config.js
  • Virtual environment di folder user (tanpa perlu akses system):
python -m venv venv venv\Scripts\activate # Windows pip install -r requirements.txt python app.py

🌐 5.5. Reverse Proxy (Nginx/Apache/IIS)

Untuk production, jalankan Flask di internal port, lalu proxy dari web server:

Contoh Nginx:

server { listen 80; server_name gudang.example.com; location / { proxy_pass http://127.0.0.1:5000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } }

Contoh IIS (URL Rewrite):

# Install ARR (Application Request Routing) dan URL Rewrite # Buat Reverse Proxy Rule → 127.0.0.1:5000

📦 5.6. Deploy ke Production (Checklist)

  • ✅ Non-aktifkan debug mode di app.py
  • ✅ Ganti secret_key dengan key yang aman & random
  • ✅ Gunakan production WSGI (waitress/gunicorn)
  • ✅ Set port sesuai environment
  • ✅ PM2 untuk auto-restart & monitoring
  • ✅ Reverse proxy (Nginx/Apache/IIS) untuk akses publik
  • ✅ Firewall — blokir akses langsung ke port Flask
  • ✅ Backup database gudang.db secara berkala

6️⃣ Cara Pengoperasian Aplikasi

🏠 Halaman Utama

Akses http://127.0.0.1:5000

  • Menampilkan 5 menu: Jenis, Satuan, Merek, Barang, Stok
  • Klik salah satu menu untuk masuk ke halaman kelola data

📋 Mengelola Data Master (Jenis / Satuan / Merek)

Ketiga halaman ini memiliki cara kerja yang sama persis:

  1. Tambah Data: Isi input "Nama" → klik tombol "Simpan"
  2. Edit Data: Klik ikon ✏️ pada baris data → ubah nama → klik "Update"
  3. Hapus Data: Klik ikon 🗑️ → konfirmasi "Yakin hapus?"
  4. 💡 Jika nama sudah ada, akan muncul pesan error duplikat.

📦 Mengelola Barang

Barang adalah kombinasi dari Jenis + Satuan + Merek:

  1. Tambah Barang: Pilih Jenis, Satuan, Merek dari dropdown → klik "Simpan"
  2. Edit Barang: Klik ✏️ → ubah pilihan dropdown → klik "Update"
  3. Hapus Barang: Klik 🗑️ → konfirmasi
  4. 💡 Pastikan data Jenis, Satuan, dan Merek sudah terisi sebelumnya.
  5. 💡 Tabel menampilkan nama relasi (bukan ID) untuk memudahkan.

📊 Mengelola Stok

Mencatat pergerakan stok barang di gudang:

  1. Tambah Stok: Pilih Barang dari dropdown → isi Jumlah → klik "Simpan"
  2. Tanggal & waktu akan terisi otomatis.
  3. Edit Stok: Klik ✏️ → ubah barang/jumlah → klik "Update"
  4. Hapus Stok: Klik 🗑️ → konfirmasi
  5. 💡 Jumlah stok harus lebih dari 0.
  6. 💡 Nama barang di tabel menampilkan format: Jenis - Merek (Satuan)

💬 Notifikasi (Flash Messages)

Setiap aksi akan menampilkan notifikasi di bagian atas halaman:

  • ✅ Hijau = Aksi berhasil
  • ❌ Merah = Aksi gagal (beserta penyebabnya)
